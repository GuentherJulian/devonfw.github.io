Script Package

The devon-ide provides a script package that is also the core. It is a simple ZIP file named devon-ide-scripts-«version».zip that you extract on your local disc.

The sources of this package can be found here and the actual content that you will get as ZIP file comes from here.

The latest stable release can be downloaded from here.

Details of scripts

create-or-update-workspace

This script will create a conf directory in root of IDE distribution, which will hold all the settings and repositories related to Maven. This will also create or update your eclipse workspace and script an according eclipse-«workspace» script, which will open eclipse with that workspace. By default «workspace» is main (if nothing else is provided as command-line argument to the script).

update-all-workspaces

You can use update-all-workspaces.bat whenever you create a new project folder in your workspaces folder to separate different project-workspaces. This update will create new Eclipse start batches allowing to run a number of Eclipse instances using different workspaces in parallel.

variables

This script holds all varialbles, which are used within entire distribution, for setting up of environment and paths.

environment-project

This script assigns all variables like maven_home, Java_home etc.

eclipse-start

This script is called when eclipse-main or eclipse-«workspace» script is being executed, which opens up eclipse.

update-eclipse-workspace-settings

This script is used for inverse merging the changes done in your main workspace back into the current settings. Basically, this script is the entry point for sharing your own eclipse settings to your project mates. After running this script you shall carefully check the diffs before committing these changes back to your VCS (e.g. git).
